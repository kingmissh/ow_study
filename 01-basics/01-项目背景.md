# OpenClaw 项目背景

## 项目演变历史

### 时间线
- **初始阶段**: Clawdbot（最初名称）
- **中期**: Moltbot（因商标问题改名）
- **当前**: OpenClaw（最终确定的名称）

### 发展里程碑
- 项目在周末 1 小时内创建的原型
- 3 周内获得 100K+ GitHub Stars
- 成为 2026 年初最受关注的开源 AI 项目之一

---

## 核心理念

### 与传统 AI 助手的区别

| 特性 | 传统 AI（ChatGPT/Claude） | OpenClaw |
|------|-------------------------|----------|
| 运行位置 | 云端服务器 | 本地设备 |
| 数据隐私 | 数据上传到云端 | 数据保留在本地 |
| 访问方式 | 浏览器/专用应用 | 消息应用（WhatsApp/Telegram等） |
| 系统访问 | 无法访问本地系统 | 完整的系统访问权限 |
| 持久化 | 有限的对话历史 | 长期记忆和上下文 |

### 设计哲学
1. **隐私优先**: 所有数据在本地处理
2. **平台无关**: 支持多种操作系统和消息平台
3. **可扩展性**: 通过 Skills 和 Plugins 扩展功能
4. **自主性**: 可以执行真实的系统任务

## 技术栈概览

### 核心技术
- **语言**: TypeScript/JavaScript
- **运行时**: Node.js
- **Agent 核心**: pi-agent-core
- **消息路由**: 自定义路由层

### 扩展系统
- **Skills**: Python/JavaScript 技能包
- **Plugins**: 模块化插件系统

## 安全性考虑

### 潜在风险
- 完整的系统访问权限
- 远程 shell 功能
- API 消耗控制
- 消息平台安全

### 安全措施
- 本地运行，数据不外传
- 可配置的权限控制
- Token 认证机制
- 审计日志
