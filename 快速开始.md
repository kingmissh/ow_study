# 🚀 快速开始 - 上传到 GitHub

## 第一步：安装 GitHub CLI

我已经为你打开了 GitHub CLI 的下载页面。

### 手动安装步骤：

1. ✅ 在打开的浏览器页面中，找到 **Assets** 部分
2. ✅ 下载 `gh_*_windows_amd64.msi` 文件（大约 10-15 MB）
3. ✅ 双击下载的 `.msi` 文件运行安装
4. ✅ 按照安装向导完成安装（使用默认选项即可）
5. ✅ 关闭当前 PowerShell，重新打开一个新的 PowerShell

---

## 第二步：验证安装

在新的 PowerShell 中运行：

```powershell
gh --version
```

如果看到版本号（如 `gh version 2.40.1`），说明安装成功！

---

## 第三步：登录 GitHub

```powershell
gh auth login
```

按照提示操作：
- 选择 `GitHub.com`（按回车）
- 选择 `HTTPS`（按回车）
- 选择 `Login with a web browser`（按回车）
- 复制显示的 8 位代码
- 按回车打开浏览器
- 在浏览器中粘贴代码并授权

---

## 第四步：上传项目

```powershell
cd E:\code\openclaw-study

# 创建仓库并上传（一条命令搞定！）
gh repo create openclaw-study --public --source=. --remote=origin --push
```

---

## 完成！🎉

上传成功后，你的仓库地址是：
```
https://github.com/YOUR_USERNAME/openclaw-study
```

---

## 如果遇到问题

### 问题 1：gh 命令不存在

**原因**：安装后没有重启 PowerShell

**解决**：关闭 PowerShell，重新打开

---

### 问题 2：无法下载安装包

**解决方案 A**：使用国内镜像
```powershell
# 使用 Scoop 安装（更快）
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
irm get.scoop.sh | iex
scoop install gh
```

**解决方案 B**：手动下载
- 访问：https://github.com/cli/cli/releases
- 找到最新版本
- 下载 Windows 安装包

---

### 问题 3：登录失败

**解决**：使用 Personal Access Token
1. 访问 https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 勾选 `repo` 权限
4. 生成并复制 token
5. 运行 `gh auth login` 时选择 "Paste an authentication token"

## 其他上传方式

如果 GitHub CLI 安装有问题，可以使用传统 Git 方式：

### 方式 1：使用我准备的脚本

```powershell
.\push-to-github.ps1 YOUR_GITHUB_USERNAME
```

### 方式 2：手动执行命令

```powershell
# 1. 在 GitHub 网页上创建仓库
#    访问 https://github.com/new
#    仓库名：openclaw-study
#    不要勾选任何初始化选项

# 2. 关联并推送
git remote add origin https://github.com/YOUR_USERNAME/openclaw-study.git
git branch -M main
git push -u origin main
```
