# 贡献指南

感谢你对 OpenClaw Study 项目的关注！

## 如何贡献

### 报告问题
- 在 Issues 中描述你遇到的问题
- 提供详细的复现步骤
- 附上相关的错误信息或截图

### 提交改进
1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的修改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

### 内容贡献

#### 文档改进
- 修正错误或不准确的内容
- 补充遗漏的信息
- 改进文档结构和可读性

#### 添加示例
- 在 `08-practice/` 目录添加实践项目
- 在 `09-source-code/` 目录添加源码分析
- 分享你的学习笔记和经验

#### 翻译
- 欢迎将文档翻译成其他语言
- 在根目录创建语言目录（如 `en/`, `ja/`）

## 内容规范

### 文档格式
- 使用 Markdown 格式
- 遵循现有的文档结构
- 添加适当的代码示例

### 代码示例
- 确保代码可以运行
- 添加必要的注释
- 遵循最佳实践

### 提交信息
- 使用清晰的提交信息
- 中文或英文均可
- 格式：`类型: 简短描述`

示例：
- `docs: 更新架构文档`
- `feat: 添加技能开发示例`
- `fix: 修正环境搭建步骤`

## 行为准则

- 尊重他人
- 保持友善和专业
- 欢迎不同的观点和经验

## 问题和讨论

- 使用 Issues 报告问题
- 使用 Discussions 进行讨论
- 在 Pull Request 中详细说明修改内容

再次感谢你的贡献！🎉
